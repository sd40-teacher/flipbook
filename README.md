# 책장 — 학교 플립북 만들기

PDF를 올리면 진짜 책처럼 넘겨 보는 플립북이 되는 무료 도구입니다. 설치나 서버 없이 HTML 파일 하나로 동작하며, GitHub Pages에 올려서 학교 구성원 누구나 쓸 수 있습니다.

## 기능

- **PDF → 플립북 변환**: 양면 스프레드, 3D 종이 넘김 애니메이션
- **종이 넘기는 소리**: 기본 내장(합성음), 원하는 mp3로 교체 가능
- **학교 로고 삽입**: 위치(4개 모서리)와 크기 조절
- **배경 오디오 삽입**: 재생/일시정지 버튼 제공
- **영상·오디오 핫스팟**: 페이지 위를 드래그해 유튜브 영상, 동영상 파일, 오디오 파일 연결
- **뷰어 내보내기**: 완성한 플립북을 단일 HTML 파일로 내보내 링크 공유
- **프로젝트 저장/불러오기**: 편집 중인 작업을 `.flipbook.json`으로 보관

## GitHub Pages에 올리기

1. GitHub에서 새 저장소(public)를 만듭니다. 예: `school-flipbook`
2. 이 저장소에 `index.html`(과 이 README)을 업로드합니다.
3. 저장소 **Settings → Pages**에서 Source를 `main` 브랜치 `/ (root)`로 설정합니다.
4. 잠시 후 `https://아이디.github.io/school-flipbook/` 주소로 접속하면 편집기가 열립니다.

## 사용 방법

1. **PDF 열기**로 학급 문집·학교 신문 등의 PDF를 불러옵니다.
2. 왼쪽 패널에서 로고, 넘김 소리, 배경 오디오를 설정합니다.
3. **핫스팟 추가 모드**를 켜고 페이지 위를 드래그한 뒤 유튜브 주소나 파일을 연결합니다.
4. **📣 게시**를 누르고 GitHub 아이디·저장소·토큰을 입력하면, 완성본이 저장소의 `books/` 폴더에 자동 업로드되고 공유 URL이 바로 만들어집니다.
   - 토큰은 GitHub → Settings → Developer settings → **Fine-grained personal access tokens**에서 만듭니다. 대상 저장소를 플립북 저장소 하나로 제한하고, 권한은 **Contents: Read and write**만 주면 됩니다.
   - 같은 파일 이름으로 다시 게시하면 기존 파일을 덮어써서 내용이 갱신됩니다.
   - 게시 후 GitHub Pages 반영까지 1분 정도 걸릴 수 있습니다.
5. 토큰 없이 쓰고 싶다면 **⬇ 내보내기**로 HTML을 내려받아 저장소에 직접 올려도 됩니다.

> 뷰어 내보내기는 GitHub Pages처럼 웹 서버에서 열었을 때 동작합니다. 컴퓨터에서 파일을 직접 더블클릭해 연 경우에는 **저장** 버튼으로 프로젝트 파일을 보관하세요.

## 조작

- 화살표 버튼, 키보드 ←/→, 모바일 스와이프로 페이지 넘김
- 핫스팟(▶/♪ 버튼) 클릭 시 미디어 재생
- ⛶ 버튼으로 전체 화면

## 팁

- 동영상 파일을 핫스팟에 직접 넣으면 내보낸 HTML 용량이 커집니다. 긴 영상은 유튜브(일부 공개 업로드)를 연결하는 방식을 추천합니다.
- PDF는 페이지 비율이 일정할 때 가장 예쁘게 표시됩니다.
